**Step 3: Create PanelContextMenu.tsx**

Component that wraps children with ContextMenu:

- `Add Panel...` — always enabled, calls onAddPanel
- Separator
- `Duplicate` — disabled (future feature, per CONTEXT.md boundaries)
- `Remove` — enabled only when panelId is provided, calls `useLayoutStore.getState().removePanel(viewId, panelId)`
- Separator
- `Reset Layout` — always enabled, calls `useLayoutStore.getState().resetLayout(viewId)`

Use shadcn ContextMenu components:

```tsx
import {
  ContextMenu,
  ContextMenuContent,
  ContextMenuItem,
  ContextMenuSeparator,
  ContextMenuTrigger,
} from '@/components/ui/context-menu';
```

Icons: use Lucide (PlusCircle, Copy, Trash2, RotateCcw) sized at 14px next to labels.

The component should be reusable — used by both PanelFrame (individual panels) and PanelGrid (empty space).

**Step 1: Wrap each panel in PanelContextMenu (PanelGrid.tsx)**

```tsx

      panelId={instance.id}

      onAddPanel={() => setAddPanelOpen(true)}
    >
      <PanelFrame ... />
    </PanelContextMenu>
  </div>
))}
```

**Step 2: Wrap the grid container for empty-space right-click (PanelGrid.tsx)**

Wrap the outer container div with PanelContextMenu (no panelId = empty space):

```tsx
<PanelContextMenu viewId={viewId} onAddPanel={() => setAddPanelOpen(true)}>
  <div ref={containerRef} className="h-full w-full">
    <Responsive ...>
      {/* panels with their own context menus */}
    </Responsive>
  </div>
</PanelContextMenu>
```

Inner panel context menus should take priority over the outer grid context menu (Radix handles this — inner ContextMenu captures the event before it bubbles to outer).
