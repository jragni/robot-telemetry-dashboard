**Conventions being enforced:**

1. One component per .tsx file — additional components become separate files
2. Types in {ComponentName}.types.ts — interfaces/types extracted from .tsx
3. Comment-described sections → sub-component files — if it needs a separator comment, it needs its own file
4. Trivially small prop interfaces (1-2 fields) MAY stay inline
5. Sub-components go as sibling files. If a sub-component accumulates its own complexity, create a subfolder.

**Critical constraint:** This is pure refactoring — NO behavior changes. Every component must render identically before and after. All existing tests must pass without modification (unless imports change).
