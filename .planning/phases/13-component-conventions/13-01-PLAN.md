**Convention rules (strict):**

1. ONE component per .tsx file — no exceptions
2. ALL interfaces/types go to {ComponentName}.types.ts — even single-field, no exceptions
3. Sub-components as flat sibling files next to parent
4. Selective barrel exports — only export sub-components if used outside parent
5. Do NOT touch shadcn/ui files (src/components/ui/\*) — they're generated
6. Pure refactoring — NO behavior changes, NO test rewrites (fix imports only)

**Types file naming:** Use PascalCase matching the parent: `PilotHud.types.ts`, `ConnectionsSidebar.types.ts`, `Header.types.ts`
