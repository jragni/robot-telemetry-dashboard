**ISS-001: ConditionalRender component**

The component should handle these common patterns:

- `{condition && <Component />}` → `<Show when={condition}><Component /></Show>`
- `{condition ? <A /> : <B />}` → `<Show when={condition} fallback={<B />}><A /></Show>`
- `{items.length > 0 && <List />}` → `<Show when={items.length > 0}><List /></Show>`

Inspired by SolidJS `<Show>` component. Prevents:

- `0 && <Component>` rendering "0" (common React gotcha)
- Inconsistent conditional patterns across codebase

**ISS-002: [].includes() for 3+ OR comparisons**

Known instances:

- `src/features/connections/hooks/useDisconnectHandler.ts:39` — `curr === 'disconnected' || curr === 'error' || curr === 'connecting'`
- `src/features/recording/export.utils.ts:67` — `value.includes(',') || value.includes('"') || value.includes('\n')`

May find more during audit.

**Rules:**

- ConditionalRender goes in `src/components/shared/`
- Replace ALL conditional rendering in .tsx files with ConditionalRender
- Replace ALL 3+ OR patterns with [].includes()
- Do NOT change behavior — just syntax
  </context>

- `when: boolean` — condition to evaluate

- `children: ReactNode` — content to show when condition is true

```
// Before:
curr === 'disconnected' || curr === 'error' || curr === 'connecting'
// After:

```

**export.utils.ts:67:**

```
// Before:
value.includes(',') || value.includes('"') || value.includes('\n')
// After:
[',', '"', '\n'].some((char) => value.includes(char))
```

Note: This one uses `.includes()` on the value (string method), not equality checks. Use `.some()` for this pattern instead.

3. Check any other matches from the audit. Only replace patterns with 3+ conditions. Leave 2-condition || alone.

Also check for 3+ AND patterns that could benefit from similar treatment, but those are rarer.
