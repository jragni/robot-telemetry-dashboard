## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage                                     |
| ----- | ----- | ----------------------------------------- |
| xs    | 4px   | Icon gaps, badge padding, grid gutters    |
| sm    | 8px   | Compact list item padding, inline spacing |
| md    | 16px  | Default element spacing, card padding     |
| lg    | 24px  | Section padding, page insets              |
| xl    | 32px  | Layout gaps between major sections        |
| 2xl   | 48px  | Hero/empty state vertical centering       |
| 3xl   | 64px  | Page-level vertical rhythm                |

**Token reference (existing in style.css):**

| CSS Token           | Value | Use                      |
| ------------------- | ----- | ------------------------ |
| `--spacing-panel`   | 12px  | Internal panel padding   |
| `--spacing-section` | 24px  | Between related sections |
| `--spacing-layout`  | 32px  | Major layout gaps        |

**Exceptions:**

- Sidebar item touch targets: 36px minimum height (py-1.5 = 6px top+bottom + 24px
  line-height) — acceptable for desktop, below 44px mobile minimum but mobile is
  out of scope this phase
- Resize handle drag zone: 4px visual, 8px hit target (via `withHandle`)
- Panel grid gutters: 4px margin (react-grid-layout `margin={[4, 4]}`) — already set,
  do not change

## Screen Contracts

### Screen 1: AppShell Layout

**Current problem:** The ResizablePanelGroup inside `<Show when={!immersiveMode}>` fails
to occupy full viewport height when `immersiveMode` is false. The sidebar appears as a
sliver because it gets zero height allocation.

**Required layout (desktop, non-immersive):**

```
┌────────────────────────────────────────────────────────────┐
│  MinimalHeader  h-12 shrink-0  border-b                    │
├────────────────────────────────────────────────────────────┤
│  ┌──────────────┬──┬──────────────────────────────────────┐│
│  │              │  │                                      ││
│  │   Sidebar    │◄►│        Main Content (Outlet)         ││
│  │  min 180px   │  │        flex-1, overflow-y-auto       ││
│  │  default     │  │                                      ││
│  │  256px       │  │                                      ││
│  │  max 400px   │  │                                      ││
│  │              │  │                                      ││
│  └──────────────┴──┴──────────────────────────────────────┘│
│  flex-1 overflow-hidden                                     │
└────────────────────────────────────────────────────────────┘
```

**Fix required in AppShell.tsx:** The `ResizablePanelGroup` must have `className="h-full
flex-1"` and the outer flex div must ensure height propagates. Current structure uses
`<Show when={!immersiveMode}>` around the entire `ResizablePanelGroup` which may not
carry the `flex-1` sizing. Ensure `ResizablePanelGroup` receives explicit `h-full`.

**Sidebar panel percentages (already set, confirm these):**

- `SIDEBAR_MIN = 14` (~180px at 1280px viewport)
- `SIDEBAR_DEFAULT = 20` (~256px at 1280px)
- `SIDEBAR_MAX = 31` (~400px at 1280px)

## Interaction Contracts

### Sidebar Resize

- Drag handle: visual `withHandle` pip (already configured via shadcn Resizable)
- Min/max: 180px–400px (existing percentages)
- No snap-to-collapse — sidebar always shows content (collapse is out of scope this phase)
- Cursor changes to `col-resize` on hover (provided by shadcn ResizableHandle)

### Robot Navigation

- Click robot card in Fleet Overview → navigate to `/robot/{robotId}`
- Click robot in sidebar → navigate to `/robot/{robotId}`
- Both use React Router `navigate()` / `NavLink` (already implemented)
- No animation on transition — instant route change

### Panel Grid

- Drag via `[data-testid='drag-handle']` grip icon only (not full panel header)
- Resize via react-grid-layout resize handles (bottom-right corner)
- Right-click panel header → context menu: "Reset to Default", "Remove Panel",
  "Tab with…" (already implemented)
- Close button (X) on non-sovereign panels: immediate remove, no confirmation
- Reset layout: no confirmation dialog — immediate reset to `DEFAULT_ROBOT_LAYOUT`

### Theme Dropdown

- Renders in sidebar footer
- Single option for now: "Default Dark"
- Setting `document.documentElement.dataset.theme` (already implemented)
- No confirmation needed — live preview on change

### Add Panel Picker

- Triggered by "Add Panel" toolbar button
- Renders as overlay `PanelPicker` (already implemented)
- Close via close button or Escape key
